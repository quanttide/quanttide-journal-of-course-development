我们现在正在尝试用这个课程工具去做 DevOps 的教程 因为我发现 我写的教程里面其实隐含了这种场景设计 但是因为没有工具把它们串在一起 所以不好整理 我突然发现 这玩意的核心作用就是把这个教程给串起来。对 这样就找到了核心的点在什么地方。这个场景 从课时里面切场景是一个难点 把场景串起来是另外一个难点。

成功验证之后，又把这个流程迁移到了我们在做的 Web Coding 入门课上，效果还蛮好的，就是结构化的。

其实关键就是套上场景：
1. 每个场景都有步骤；
2. 场景之间会有关系；
3. 把它画成正常和异常的就可以了，这已经足够了。

因为我们有这种表达需求 才会产生这种结构需求 从原始的教程出发去整理 我发现反过来也可以去整理这个教程 我觉得这个思路很清楚。

他需要以教程、手册或文档一类的资源，去生成这个结构。但是，只要有一个已经表达出来的东西，就可以把它提取出来。

把里面隐式的这种逻辑提出来，显性化。如果看不对，人类再改就可以，这样脚本就非常清楚。

这样其实也梳理了 DevOps 的文档 我们用 DevOps 和课程两个进行“左脚踩右脚” 一下就把它给迭代出来了 我觉得这是很巧妙的一件事情。

而且这样客户端也就有目标了 无非就是一大堆的编排工具嘛 按照这个类似的方法做就可以了。

大概想象了一下 比如课程会有一个编排工具 课时会有一个编排页面 场景也会有一个编排页面 无非就是把课程录制放进去就可以了。
